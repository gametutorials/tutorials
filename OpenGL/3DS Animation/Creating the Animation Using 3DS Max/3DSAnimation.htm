This page is designed to be an aid for creating animation in 3D Studio Max.  Many of you might already know how to model and set key frames in Max, but I am going to show you how to make it work so it animates correctly in the 3DS Animation Tutorial created by www.GameTutorials.com.  This type of animation is just the beginning before you move into bone animation.  There are a lot of restrictions and loss of true motion when doing it this way.  Basically, you are just creating objects and then translating, scaling or rotation them with key frames.  That means you can not use bones to give the object of character life like motion.  This is how it was done before skeletal animation was used in the older games.


Let me start by saying that the version of max that we will be using is 3D Studio Max 3.0.  The differences between 3 and 4 aren't big enough to max 2 separate versions of this tutorial.  If you are using 2.5 and below it might be a tad bit different, but if you already know 3DS Max it shouldn't be a problem.  Below you can see 3D Studio Max open with the 4 default views (Figure 1.1).  We will be working with the bottom right view called the perspective view.  If you click on that view port, then click the Min/Max Toggle button in the bottom right it will allow you to just see the perspective view, which will give you a larger window to work with. You can click the button again to return to the 4 view ports.

 
Figure 1.1

First off, let's do a quick walk through on how to create objects in 3DS max.  Let's start by creating a simple box.  Click on the Create tab on the right.  You should now see for some of the object types listed a Box, Cone, Sphere, etc (Figure 1.2).   If you don't, make sure you have the Standard Primitives selected from the drop down menu and the Geometry button is selected right above that.  Click on the button that says Box.  A section below should appear that allows you to set the new box's attributes like length, width and height, along with the amount of segments (detail) it will have.  Do not set this data yet, but first put your cursor in the perspective view port around the origin (0, 0, 0) and click, hold, then drag the mouse to the right and up a bit.  This will create the length and width of the box.  You will notice, once you let go it will then create the height depending on where you move the mouse.  If you move it up it will increase the height, if you move it down, it will decrease the height.  When you found a decent height click the mouse and it  

Figure 1.2

will save your box.  The width and height doesn't matter right now, nor the position of the box because we will set the values in a second.  The box should have a random color that 3DS Max set for you (but remember this isn't a material!).  Refer to Figure 1.3 for an example of what yours might look like.  If you look over to the right you will notice that the Length, Width and Height values reflect the box you just created.  They are most likely very crude and precise numbers.  Let's make this a cube by setting each of those values to 30.  Now we should have a cube with a size of Length, Width and Height of 30.  You might notice up above there is an option for the creation method to create a cube instead of a box, but I wanted you to get comfortable with inputting your own values.

Figure 1.3

 

You can you use the Arc Rotate button and the Zoom button to rotate and zoom around the world to get a better view of the object you just created.  To zoom in, you can also use the mouse wheel on your mouse if you have one.  If you notice there is a local (X, Y, Z) axis in red centered at the bottom of the cube.  That is the object's Pivot Point.  We will discuss later on how to change that.  Basically, a pivot point is the origin (0, 0, 0) for the object in model space.  We are looking at world space, but each object has their own model space.  So when you translate that object to (0, 0, 0) it's pivot point will be at (0, 0, 0) in world space.  Then however the object is position according to it's pivot point it will then be translated.  For instance, right now the cube's pivot point is centered at the bottom of the cube.  If we translated the cube to (0, 0, 0) it would be above the origin, but centered along the X and Y axis (Z is up in 3DS Max) and the pivot point would be at (0, 0, 0) in world space.  Let's say we move the pivot point to the center of the cube.  Now, if we translated the cube to (0, 0, 0) in world space the pivot point would still be at (0, 0, 0), but the cube would be totally centered around the origin.  Basically, where ever the pivot point goes, the object goes and keeps the same position from it.

Speaking of translating, let's move the object and center it at the origin (of course on top of the grid because its pivot point is at the bottom of the cube).  To do this, we want to make sure the cube is selected, then click on the Translate button .

Once we selected this button we want to right click on it.  This will bring up a dialog box (Figure 1.4) that will allow us to numerically translate the object to a specific (x, y, z).  Type in 0 for the X Y and Z Absolute:World values.  You should see that your cube is now centered around the origin but on top of the grid.


Figure 1.4

If you don't want to type in where you want the box to go, you can select the Translate button and click and drag the box to where you want.  You can restrict the movement along certain axis' by clicking on one of the Restrict buttons .  On the right of those there is also a button menu to choose multiple axis's.  These work the same for rotations too.  Try playing around with some of the other shapes besides boxes and get comfortable with moving them around.  Before we go on, let's assign a material to our object.  I won't go into how to put a texture on the object here, but I will give you the jist of how to do it.  If you hit 'm' it will bring up the Material Editor (Figure 1.5).  

Select a color from the material editor.  Then click again and drag the material onto the object.  Now the object will have that color mapped on it.  You should at least have a color material on all of your objects, otherwise they will appear gray in the 3DS Loader.  If you want to apply a texture map, I will quickly explain how.  Click on the gray button next to the diffuse color in the editor.  A dialog box should come up with some options for a map, double click the bitmap option.  It will then allow you to choose the file that will be the texture map.  After you choose the file you should see the texture mapped around the color sphere you had selected in the material editor (usually the first one).  Now, click and drag that texture onto the object.  Then, you will need to click the Show Map In View Port button .  This will display the texture map in your perspective view port.  You can then from there apply a UVW Map in the Modifier tab next to the Create tab.  There you can mess with positioning it. 

Figure 1.5

 

 

 


Now that you have the basics (or already did) of creating and moving around shapes in 3DS Max, let's get on to what you want to know, animation!  Let me explain again on how this type of animation works.  We aren't doing bone animation, just key frame object animation.  If you wanted to animation an object going from one place to another in a straight line or along a curve, you don't want to move the object every frame for say 60 frames do you?  With key frames, you can just give the object a start and end location and have the rest done for you.  If you want it to move along a curve, just position the object at variables points along that curve and it will interpolate the rest for you.  The more key frames though, the better the accuracy the animation.  So, let's try to do a simple animation.  We will attempt to move our cube that we made from one position to another, say (0, 0, 0) to (10, 10, 0), assuming Z is up.  Since our object is already at (0, 0, 0) we just need to set a key frame for (10, 10, 0).  To do this, click on the Animate button .  This will turn red, along with the view port that is selected.  Ours happens to be the perspective view port.  Now we are ready to set key frames.  The first frame (0) just happens to set a key frame by default.  Notice the animation bar .  It says that we are on frame 0 with 100 frames total in the animation.  This is the default that 3DS Max gives us.  You can click and drag the animation bar left and right to scroll through the different frames.  You will notice that the frames currently are all the same because we haven't done any animation yet.  Let's leave the animation frames to 100 for now, and I will show you later how to change them.  Scroll the animation bar to the very last frame 100.  Now, with the Animate button still selected move the box to (10, 10, 0) by right clicking on the Translate button and manually typing in (10, 10, 0).  You can also move it there yourself if you want to by clicking and dragging the box there.  Now, that's all you need to do to set the key frame.  With the Animate button set, it remembers what the position, scale and rotation the object has for every frame you go to and change.  If you want to view your animation click the Play button or use the Animation bar to scroll through every frame by dragging it left and right.  To stop the animation, click the Play button again (It will turn to a stop symbol).  That's it!  You just did your first key frame animation.  Now try going to frame 50 and moving the box to another play.  Make sure you have the Animate button down or else it won't compute your changes for the animation.  You will notice now that if you move the box in a different place that doesn't move in a straight line there is a curve to it's movement.  Try doing some key frames using the Scale and Rotation buttons.  If you want to move the object somewhere and then have it interpolate back, just set the first and last frame with the same translation values.

 

Now that we have the basics of doing key frames, let make a simple animation that could load into our program.  We will make a simple character that has a basic walk cycle.  It might look like Frankenstein but it will prove as an easy example anyone can do.  We start with the body.  First, create a Cylinder with a Radius of 6 and Height of 20.  You can find the Cylinder button 2 below the Box button in the Create tab (See Basics above).  Leave the rest of the data for the cylinder default.  Next, translate the object to (0, 0, 20).  Open the Material Editor (hit 'm') and give the body a light blue color.  Your scene should look something like  Figure 2.1


Figure 2.1

Next, we want to create the head. Create a cone with Radius 1: 6, Radius 2: 4 and a Height of 8.  The Cone button can be found on the right of the Box button.  Translate the head to (0, 0, 42).  It should now be positioned a bit above the body.  Once again, open the Material Editor and select the skin color material.  Drag it onto the head to apply it.  You can find this color if you scroll all the way to the right and down about half way.  Your scene should look something like Figure 2.2.

Figure 2.2

Now we need to add the arms to our character.  Create another cylinder with a Radius of 2.5 and a Height of 20.  Translate the arm to (9, 0, 20).  There should be an arm on the character now.  We need to create another, but we don't have to create a new cylinder.  Select the arm you just made.  Click on the Translate button .  Hold down shift and click and drag the arm to another location and left go.  You will notice that it brings up a dialog box that asks if you want to create a copy, instance or reference.  Leave it at the default "copy" and hit the OK button.  Now we have another arm.  Translate that arm to (-9, 0, 20).  Apply the same skin material to the arms are you did to the head.  Your character should look something like Figure 2.3.

Figure 2.3

 

Last but not least, we need to model the legs.  Create another cylinder with a Radius of 3.5 and a Height of 20.  Translate the leg to (3.5, 0,20).  There should be a leg on the character now.  Copy the other leg the same way you did the other arm and translate it to (-3.5, 0, 0).  Assign a darker blue material to the legs.  Now you should be done modeling the character.  It should look something like Figure 2.4.  Before we start animation the model, we want to set the arm's and leg's  pivot points.  Since we need to rotate the legs and arms to simulate a walk, if we rotated them around the current pivot point it wouldn't look right.  We want them to rotate from the top of the cylinder.  Let's discuss how to do that next.



Figure 2.4


Select the a leg.  Click the Hierarchy Tab on the right of the Modify Tab.  It should bring up a bunch of options for Pivot, IK and Link Info (Figure 2.5).  Click the Affect Pivot Only button.  It should turn a different color.  This means that what ever we do, be it translations or rotations, it will just effect the pivot point of the object, not the object itself.  We want to move the pivot point near the top of the leg so it rotates correctly.  Translate the leg's pivot point Z value to 18.  Do the same thing for the other leg.  Now we'll do the arms.  Translate both arm's pivot point Z values to 38.  Now the arm's and the leg's pivot points should be near the top of the cylinder.  This will allow us to rotate around a more appropriate axis.  If you click and drag your mouse over all of the objects to select them and then turn on the Affect Pivot Only button, the pivot points should be in the same position as shown in Figure 2.6.  You can unclick the Affect Pivot Only button now.

Figure 2.5

Figure 2.6

We are ready to now go on to moving the arms and legs around.  The character's walk cycle will be more like a march in place animation.  It will be a seamless animation though.  To start animation, let's move the Animation Bar to Frame 0 if it isn't already.  Click the Animate button to allow key frames to be set. First we will do the legs.  Now that the Animate button is set, move the Frame Bar to frame 25.  Click on the Rotate button next to the Translate button .  Select the right leg of the character (his true right leg, our left side).  Now, right click on the Rotate button.  It should bring up a dialog box similar to the Translate Dialog Box.  Look at Figure 2.7 for the Rotate Dialog Box.

Figure 2.7

There should be all 0's in the fields as shown above.  Now, we want to rotate the current leg along the X Axis.  To do this, we can just type in -45 in the Absolute:World X field.  Make sure that was set by clicking in another field or pressing enter.  Now close the dialog box.  You will notice that the characters right leg has lifted off the ground as if to step forward (Figure 2.8).  If the rotation did not happen correctly, make sure the pivot point was set right.  It should be located near the top of the leg.

Figure 2.8

If you scroll the Frame Bar back and forth between frames 0 to 25 you should see the leg rise and fall.  If you do not see this make sure you have the Animate button selected and your view port should have a red outline to it.  Now move the Frame Bar to frame 50.  Right click again on the Rotate button and set the X rotation back to 0 again.  If you move the Frame Bar back and forth now, it should animate the leg moving up and then back down again.

To animate the other leg, we need to do something a tad bit different.  Since this character will be doing a march, we want the right leg to go up and come down, then the left leg to go up and come down.  That means that we will want to set key frames for the left leg at frame 75 and at frame 100.  The problem with this is, if you just set a key frame at 75 for the left leg, it will interpolate from 0 to 75.  We need to set a key frame a 0 and then 50 first.  To do this, click on the left leg and move the Frame Bar to frame 0.  Right click on the Frame Bar.  A dialog box will pop ups that asks if you want to create a key frame on the current frame.  Click OK and you will notice a gray circle appear underneath the Frame Bar.  That means that a key frame is set there.  Now go to frame 50 and do the same thing.  You should see a gray circle under frame 0 and frame 50.  If you go back to the right foot it will also have one of these gray circles for frame 25 and 50.  You might be thinking that you should just have to do it at frame 50, but when you export the model the way it handles the animation it won't work.  It would actually be a good idea to set a key frame at frame 0 like this for every object that you manipulate just to make sure.

So, once you set the key frame for the left foot at frame 50, go to frame 75.  Do the same rotation that you did for the right foot (-45 degrees along the X axis).  Next, move the Frame Bar to the very last frame, 100.  Rotate the left leg back to it's original position by setting the X axis rotation from -45 to 0 again.  Press the Play button  to view the animation or move the Frame Bar back and forth.  The character should appear to be doing a march, with the right foot forward, down, then the left foot forward, then down.  Remember to make sure you have the Animate button down while you are setting the key frames or else it will not work.

Simple animation huh?  Well let's make it a bit more life like and move his arms.  Select the character's right arm.  Go to frame 25.  Rotate the arm by 45 degrees along the X axis.  The arm should now be rotating backwards if you want the animation.  Make sure it looks similar to Figure 2.9.

Figure 2.9


Now go to frame 75.  Rotate the arm by -45 degrees on the X axis.  Lastly, go to frame 100 and set the rotation to 0.  The arm should now be resting at the character's side.  If you want the other arm to move, select the left arm and do the same thing but do the opposite rotations.  First rotate by -45 degrees along the X at frame 25, and then at frame 75 rotate it by 45 degrees along the X.  Finally, at frame 100 set it back to 0 for the X.  If you press play now you should have a character doing a simple march animation with his arms going back and forth.  If you go to frame 75 your character should look like Figure 2.10.

 

Figure 2.10


That's it!  That is how you do a simple character animation using key frames.  See if you can get the head to bounce while he marches.  I dare ya. 

Once you have the animation the way you want it, turn off the Animate button and move the Frame Bar to frame 0.  Go to the File menu and choose Export..., Then obviously save it as a .3ds file.  That should be good enough for you to load it into your program and use our key frame animation tutorial to play the animation back. 

If you want to change the amount of frames your animation is, instead of the default 100, Right Click on the Key Mode Toggle button.  A dialog box will pop up with options to change the frames per second speed, as well as many other options.  They are pretty self explanatory.  A good frame size is 60.

In closing, I would like to say that there may a few things you have to mess around with to get the animation working correctly.  There has not been extensive testing on the animation loader so there may be some times it might do something funny.  It is not intended to be a robust animation loader but to teach the basics of doing simple key frame animation from 3D Studio Max.
